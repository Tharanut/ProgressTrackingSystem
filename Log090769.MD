## 7. ปัญหา Push ขึ้น GitHub (ยังไม่จบ)

- Remote: `https://github.com/Tharanut/ProgressTrackingSystem.git`
- ครั้งที่ 1: push ด้วย credential ที่ cache ไว้ในเครื่อง (account `payakkamron`) → **403 denied to payakkamron**
- ครั้งที่ 2: ผู้ใช้ให้ Personal Access Token (`ghp_mE2u...`) → ลองฝัง token ตรงใน URL คำสั่ง push → **ถูกระบบบล็อก** (token จะไปโผล่ใน shell history/process list)
- ครั้งที่ 3: ใช้ token ผ่าน env var + inline git credential helper (ไม่ฝัง token ตรงในคำสั่ง push) → แก้ปัญหาการฝัง secret ได้ แต่ credential manager ของ Windows (GCM) ตอบกลับด้วย credential เก่าก่อน (ยัง denied to payakkamron)
- ครั้งที่ 4: เคลียร์ credential helper เดิมด้วย `-c credential.helper=` ก่อน แล้วค่อยใส่ helper ใหม่ → คราวนี้ authenticate เป็น **Tharanut** ถูกต้อง แต่ยัง **403 — denied to Tharanut** (token ไม่มีสิทธิ์เขียน repo นี้ อาจขาด scope `repo` หรือ fine-grained token ไม่ได้ให้สิทธิ์ repo นี้)
- ครั้งที่ 5: ผู้ใช้ยืนยัน token เดิม + ชื่อ account `tharanut` → กำลังจะลอง push ซ้ำ แต่ผู้ใช้ **ยกเลิกคำสั่งนี้** (ขอให้บันทึก log นี้ก่อน)

**สถานะปัจจุบัน:** โค้ดทั้งหมด commit ไว้ในเครื่อง (local repo, `main` branch, working tree clean) พร้อม push แต่ **ยังไม่ได้ขึ้น GitHub** — ต้องตรวจสอบสิทธิ์ของ Personal Access Token ที่ https://github.com/settings/tokens ว่ามี scope `repo` (classic token) หรือถูกให้สิทธิ์เข้าถึง `Tharanut/ProgressTrackingSystem` (fine-grained token) หรือยัง
