## 4. เพิ่มข้อมูลตัวอย่าง (Sample Data) — 20 คน + 2 Project

- ตอนนั้นยังไม่มีหน้าจอ Employees ในแอป จึงเพิ่มข้อมูลตรงผ่าน Supabase MCP (`execute_sql`)
- **20 พนักงานใหม่**: สร้างจริงเป็น Supabase Auth user (ผ่าน `auth.users` + `auth.identities` โดยตรง,
  bcrypt hash เดียวกับรหัสผ่าน seed เดิม `ProjectPulse#2026`) ให้ trigger `handle_new_user` สร้าง
  `profiles` ให้อัตโนมัติ แล้ว UPDATE เติม `employee_code`/`department_id`/`cost_rate_per_day` — กระจาย
  role (2 pm, 2 management, 1 viewer, 15 member) และแผนก (IT/DEV)
- **2 Project ใหม่** (`PRJ-003` Mobile Expense Tracker, `PRJ-004` IT Asset Management System) พร้อม
  project_members, tasks (4 ต่อ project), time_logs — ให้ trigger คำนวณ Actual Man-Day/Cost/
  Progress % อัตโนมัติ (ตรวจสอบผลลัพธ์ตรงกับการคำนวณมือ 100%)
- **เจอปัญหาระหว่างทาง**: ตั้ง id ให้ tasks ชนกับ id ของ seed data เดิม (`d0000000-…-0001..0004` ถูกใช้
  ไปแล้วตั้งแต่ migration `03_seed_sample_data`) — transaction rollback ทั้งหมดอัตโนมัติ (ไม่มีอะไรเสียหาย)
  แก้โดยเปลี่ยนไปใช้ prefix `e0000000-…` แทน แล้วรันใหม่สำเร็จ

## 5. หน้าจอ Employees — เพิ่ม/ลด/แก้ไขพนักงาน

- พบว่าเดิมไม่มีหน้าจอสร้างพนักงานในแอปเลย (`profiles` ถูกสร้างผ่าน trigger เท่านั้น) — สร้างฟีเจอร์ใหม่:
  - `src/lib/supabase/admin.ts` — service-role client (สำหรับ `auth.admin.createUser`)
  - `src/lib/actions/employees.ts` — `createEmployee` / `updateEmployee` / `setEmployeeActive`
    (จำกัดสิทธิ์เฉพาะ `admin` ให้ตรงกับ RLS policy จริงของตาราง `profiles`)
  - ฟอร์ม `NewEmployeeForm` / `EditEmployeeForm` / `ToggleEmployeeActiveButton`
  - หน้า `/employees` (list + create) และ `/employees/[id]` (edit + deactivate)
  - `AppNav.tsx` เพิ่มเมนู "Employees" (แสดงเฉพาะ role admin)
- ออกแบบ "ลบพนักงาน" เป็น **soft-delete** (`is_active = false`) แทนการลบจริง เพราะการลบ profile จริง
  จะ cascade ลบ `time_logs`/`project_members` ของคนนั้นทิ้งไปด้วย (ตรวจ FK constraint ยืนยันแล้ว)
- **ปัญหา Service Role Key**: `.env.local` มีตัวแปรอยู่แต่ค่าว่าง (comment เดิมบอกไว้ว่า "Fill in when
  needed") — ต้องหยุดพัฒนาและแจ้งผู้ใช้ให้ไปคัดลอกค่าจาก Supabase Dashboard
  - ผู้ใช้ใส่ค่าครั้งที่ 1: เป็น **Project ID** ไม่ใช่ key
  - ผู้ใช้ใส่ค่าครั้งที่ 2: เป็น **anon/public key** ตัวเดิม (decode JWT payload เจอ `role: "anon"`) —
    `auth.admin.createUser` ปฏิเสธด้วย "User not allowed"
  - ผู้ใช้ใส่ค่าครั้งที่ 3: ถูกต้อง (`role: "service_role"` ใน JWT payload) — รีสตาร์ท dev server
    (ต้อง kill process เดิมที่ค้าง port 3000 ก่อน ไม่งั้น Next.js จะไม่รับ env ใหม่)
- ทดสอบ end-to-end จริงกับ Supabase project (เพราะไม่มี `chromium-cli`/Playwright ในเครื่องนี้ สำหรับ
  driving browser จริง) ผ่านสคริปต์ Node เรียก path เดียวกับที่ Server Action ใช้ — **ผ่านครบ 12/12**:
  sign in admin, สร้าง user จริง, trigger สร้าง profile, enrich ข้อมูล, แก้ไขชื่อ, deactivate/reactivate,
  และยืนยันว่า role `pm` ถูก RLS บล็อกไม่ให้แก้ไข profiles (0 rows, ไม่ throw error) — ลบ test user ทิ้งหลังทดสอบเสร็จ
